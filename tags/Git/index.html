Git 強行將 origin/master 退回某個 Commit

目的

  • 將 Origin/master 退回到一開始的 Commit
    • 會這麼做的原因是一開始使用了 master 分支做開發，後續為了將”開發中”與”實際運行”的 Code 做分支管理，所以棄用了 master 分支，而另外 brach 了其他分支，所以我想將 master 和 origin/master 都退回最初的 Commit

步驟

origin/master 分支是表示遠程儲存庫上的 master 分支，並不代表本地儲存庫的指針，以下方法是強制 PUSH本地的更改然後更新到遠程儲存庫上

這個步驟會直接影響 Remote 的 master 務必下手前要三思

問題

Q1: 不允許強制推 Code 到受保護的分支

  • 如果自己是這個 repo 的 master 那可以透過，GitLab 上面先將保護分支的規則關閉，然後退回想到退的 commit 後再將規則加回去
