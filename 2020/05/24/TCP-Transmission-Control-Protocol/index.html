TCP (Transmission Control Protocol)
Created 2020-05-24| Updated 2020-05-24
Post View:

前言

TCP (Transmission Control Protocol) 是 OSI 7-Layer 裡面 Layer 4 傳輸層的協議

為什麼需要學習 TCP ?

TCP 是目前網路傳輸中不可或缺而且使用頻率最高的一個協議, 以前網路實體線路(例如: RJ-45, 同軸電纜….)的技術沒有現在這麼好, 經常會因為物理的因素而導致網路傳輸過程中發生掉包或是亂序的問題, 而 TCP 因此而生, TCP 協議確保了資料的完整傳輸並且針對傳輸過程中, 所發生的掉包或亂序,設計了一套相對應的解決辦法,而在本篇將會針對這些機制進行解說

TCP 簡介

  • TCP 是一個 End-to-End (端點對端點) 的傳輸協定, 這樣說很難明白對吧 ?
    • End-to-End 的意思是指,只要我兩端的人溝通過講好, 那我們不需要管中間經過的每個人的意見
    • Host A 和 Host B 中間不管經過任何 Router 或 Switch 都不影響 TCP 的協議, 如圖

  • 在傳輸層(Layer 4) 提供兩種類型的傳輸協定, 一種是連接導向,另一種是非連接導向, 底下我將對這兩種類型說明

連接導向（Connection-Oriented )

傳輸層的連接導向協議是 TCP ,連接導向是指假設終端的兩端要傳送資料, 則必須先建立一條虛擬的連線(非實體), 才能開始傳送遠方的終端, 而本章說明的 TCP 就是個連接導向的協定, 為什麼需要建立連線呢 ? 因為 TCP 必須保證接收端有真的收到資料, 並且假設兩端的設備等級懸殊過大,導致一方的能力無法負荷, 所以先溝通建立連結就可以知道對方能夠收多少資料或以多快速率去傳輸資料,就跟打電話是一樣的模式, 一定是要先撥號然後對方接起來,建立連線後才能開始說話

非連接導向（Connectionless）

傳輸層的非連接導向協議是 UDP (本篇不做介紹), 非連接導向在傳輸資料前並不需要建立連接, 傳送端想傳就直接傳, 而接收端也不知道何時會接收到資料, 非連接導向不會確認接收方是否存在, 就算對方無法接收資料仍然可以將資料傳送出去, 那大家都應該會有個想法就是, 那大家都使用 TCP 來傳輸就好了啊 ! 最保險的方式, 但是 UDP 卻還是有它的好處存在例如, 可以省去一些建立連線的步驟與確認的回覆,相對的傳送速度就會比較快, 所以需要使用哪一種導向協議則需要考慮是傳送哪類型的資料再做決定
