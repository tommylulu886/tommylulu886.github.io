前言

Ethernet （中文：乙太網路）是我進到目前公司的第一個課程, 那我覺得它會是一個踏入網路世界很好的一個開端, 為什麼這樣說呢 ? 網路對大部分人而言是一個抽象的東西吧, 雖然現今大家每天都活在網路世界裡 (((誤… , 但是對我而言它不難懂然後也可以用一些好懂的方式理解, 例如圖解或是比喻法來協助理解這些理論, 所以別擔心該從哪裡入門, 讓我們就從現在還有這篇文章開始吧 !

正言

這邊為了讓大家更有深刻的了解 Ethernet , 所以來考古吧 !

Ethernet 歷史

Ethernet 是一種區域網路技術。由 IEEE 802.3 標準制定了乙太網路的標準，它規定了包括實體層的連線、訊號和媒介存取層協定的內容。

乙太網路 (Ethernet) 是 Xerox 在 1970 年所發展的封包交換 LAN 技術, Xerox 和 Intel 和 Digital equipment 在 1978 年一同訂定 Ethernet 的標準, IEEE 也釋出編號 802.3 的標準規格, 回到現今來看 Ethernet 已經是不可或缺的重要網路技術, 從企業到大眾所使用的各個網路環境中都有它的存在,而為了符合市場需要,在效能和傳輸距離方面都越來越好，目前最高規格已經到了 400G 的傳輸速度。

  • 下圖為 Ethernet 在整個 TCP/IP Protocol Suite 和 OSI 七層中相對應的位置, 也是本篇的解說範圍

Ethernet 命名規則

  • 區域網路常以 M Base(Broad) N 表示

    • M 代表傳輸速度, 其單位為 Mbps 　
    • 中間英文字 Base 代表基頻 (Baseband) 傳輸的意思 , 若是 Broad 則代表是寬頻 (Broadband) 傳輸的意思
    • N為數字或英文，若英文字代表使用線材種類，數字則代表每區段纜線的最長傳輸距離。

    範例

    1. Ex: 100BaseF，「 F 」代表使用線材為光纖 Fiber。
    2. Ex: 10Base-5，5代表纜線最長傳輸距離為500公尺。
    3. Ex:10baseT 速度 10Mbps 以基頻傳輸，「 T 」代表使用線材為雙絞線 RJ-45

Physical Layer (L1) 實體層

同軸電纜

  • 在最一開始的乙太網路設計都是使用這種同軸電纜 (Coaxial cable) ,那它是電線也是傳輸線, 主要是由四層不同材質包裹而成, 每頭的末端都需要一個電阻來防止電波反射, 電纜本身是完全被動的, 所有網路功能的主動權都在電腦中

  • 最早的販售的 Ethernet 為 10Base-5 (IEEE 所制定的規格), 也是所謂的粗同軸纜線 RG-11 (Thick Ethernet) ,每個區段 (Segment) 最大長度為 500m , 每個區段最大的節點 100 nodes, 在電腦和同軸電纜之間會使用收發器 (Transceiver) 裝置來進行實體的連接, 然後再用 AUI 傳輸線跟 PC 做連接

一個區段的節點只能有 100 個, 換句話說就是一個區段同軸電纜只能接 100 台電腦

  • 收發器 Transceiver 的用途
    • 偵測和控制電子訊號的類比硬體
    • 偵測是否有裝置正在使用線路中
    • 可以將類比訊號轉為數位訊號

雙絞線

拓樸介紹 Topology

拓樸的意思是網路實體連接的結構形式

  • Bus Topology 匯流排拓譜
    • 此拓樸主要是同軸電纜在使用的, Medium 兩邊終端都會有電阻讓訊號送到終端時消失
    • 同一時間只允許一個節點 (node) 傳送資料
    • 採用廣播方式傳送資料（傳送給所有節點 )
    • 不易找出問題點，線使用的少，一條線可接100個節點，任一段線路故障，則會造成網路中斷

範例 : 10Base5、10Base2

  • Star Topology 星型拓樸

    • 此拓樸主要是使用雙絞線
    • 需要使用交換器 (Switch) 或集線器 (Hub) 來連接電腦
    • 任一個節點傳送資料都必須經過中間那個交換器, 由交換器轉發資料
    • 只有中央交換器故障時, 整個網路才會癱瘓

    範例 : 10BaseT、100BaseTX

  • Ring Topology 環狀拓樸
    • 此拓樸使用雙絞線、同軸電纜、光纖 (Fiber) 都可
    • 運作方式是拿到記號 (token) 封包的擁有傳送的權利,這種拓樸好處是不會發生碰撞, 而且每個節點都擁有平等的傳送權
    • 但在網路流量正常時則無法發揮 100% 的實力, 因為沒拿到記號 (token) 封包的節點都不能傳資料, 但早期還是有另外同時循環多個記號的方法,用來提高網路效能

範例 : Token Ring 、FDDI（Fiber Distributed Data Interface）

圖片表示 A 原本有記號 (Token) 然後 A 傳完資料後, 傳給下一位 B 讓它傳送資料, 然後 B 也傳完後依序傳給下一位 C

Data Link Layer (L2) 資料連結層

Data Link Layer 主要在制定以物理層電纜(光纖、雙絞線…等等)所連接裝置的通訊規格, 而在物理層所處理的是將接收到的 0、1 這種數位訊號, 轉換成 L2 所處理的訊框 (Frame) 也有人叫它”幀”, 在詳解 Frame 的每個細節前, 你可以想像訊框 (Frame) 就像是寫信給朋友一樣, 我寫給朋友的內容是重點, 但除了內容以外我們也必須寄信前給它一個信封包裹它, 而且信封外面要寫上寄件人名字、寄件人地址、收件人名字、收件人地址, 郵差才能幫你寄信對吧 !

所以 Frame 就像信封一樣記錄這些重要的資訊然後透過這些資訊幫你寄給對的人 !

MAC

  • IEEE 802.3 規範將 Data Link Layer 畫分成兩塊子層
    1. LLC, Logical Link Control 邏輯鏈路控制
    2. MAC, Medium Access Control 媒體訪問控制

通訊方式

根據傳輸的對象(數量)和行為,通訊的方式有分成幾種,這邊會依序介紹,個人認為這個通訊方式非常重要,務必確定了解清楚每個方式的差異在哪再繼續往下看

  1. 單播 (Unicast)
  2. 廣播 (Broadcast)
  3. 群播 (Multicast)

單播 (Unicast)

  • 從字面上來看 Uni 具有單一的意思, 然後 Cast 具有投射、投放的意思
  • 例如: 湯米把信寄給楷楷
  • 一對一的通訊

圖例: PC1 發送資料給 PC2

廣播 (Broadcast)

  • 從字面上來看 Broad 具有寬闊的、寬廣的的意思, 然後 Cast 具有投射、投放的意思
  • 例如: 湯米結婚把喜帖寄給全部認識的朋友
  • 一對全部的通訊 (在 Broadcast domain 廣播領域內的所有 Host)
  • 廣播領域就是廣播可以傳送的範圍

圖例: PC1 發送資料給這個 Broadcast domain 廣播領域的全部人,裡面包含 PC2、PC3

群播 (Multicast)

  • 從字面上來看 Multi 是英文單字 multiple 具有多個、多種的意思, 然後 Cast 具有投射、投放的意思
  • 例如: 湯米結婚只把喜帖寄給幾個比較好的朋友而已
  • 一對某個特定群組(複數)的通訊

圖例: PC1 發送資料給一個幾個需要這個資料的群組,群組裡面包含 PC2、PC4 , 而 PC4 用不到這個資料所以沒有把它加進群組,也就收不到此資料

CSMA/CD

  • 全名（Carrier Sense Multiple Access/Collision Detect)
  • 當同一個區段內有任何一台主機需要傳輸資料時,要先聽(Listen) 同個區段內有無工作站也正在傳輸資料, 若有主機正在傳輸資料時, 則必須等待對方傳輸完畢後才能接著傳輸,
