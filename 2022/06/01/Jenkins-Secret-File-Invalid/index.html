The GitLab authentication of Jenkins agent failed - Key Invalid Format

Situation

Jenkins Job 觸發 Jenkins Agent 去與 Gitlab 做 Authentication 時發生 SSH Key (.id_rsa) Format Invalid 的錯誤

Task

由於 SSH Key 原先在匯入 Jenkins Controller 時，是使用 Windows 電腦然後複製貼上到 Key File 內的，這會導致因為複製貼上所產生的換行符號不同，讓 GitLab 認為是使用錯誤的格式，故所以要改成使用 Key File 直接傳遞，就可以避免無法識別的問題

使用複製貼上時 Windows 文本編輯器會幫我將 EOL 轉換為 CR LF

Action

⚠️ 請勿將 SSH Key 直接以複製的方式，傳遞給實做的工程師
  1. 在 SSH Client 端先將 SSH Key 匯出來
  2. 登入 Jenkins Controller 內，並新增一個 Credential
  3. Authentication Kind 選擇 Secret File
  4. 並將剛剛匯出來的 SSH Key 匯入 Jenkins Controller 內

  1. 將已經匯出的 SSH Key File 匯入 Jenkins 內
    • 無需輸入副檔名
